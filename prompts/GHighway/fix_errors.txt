### 7. Load / Store errors — alignment and signature

**Alignment crashes (segfault at runtime, not a compile error but critical):**

`hn::Load` / `hn::Store` require the pointer to be aligned to `HWY_ALIGNMENT` (32 bytes on AVX2, 64 bytes on AVX-512). Memory from `std::vector`, `new[]`, `malloc`, or unknown function parameters is **NOT** guaranteed aligned.

**Fix:** replace `hn::Load` → `hn::LoadU` and `hn::Store` → `hn::StoreU` unless the buffer was allocated with `hwy::AllocateAligned<T>()` or declared with `HWY_ALIGN`/`alignas(HWY_ALIGNMENT)`.

**Signature errors:**

- `hn::LoadU(d, ptr)` — `(tag, const T* pointer)`.
- `hn::StoreU(vec, d, ptr)` — `(vector, tag, T* pointer)`. Argument order is **(vector, tag, pointer)**, not (tag, vector, pointer).
- The pointer type **must match** the tag's element type. `hn::LoadU(d_float, int_ptr)` is a type error — cast or use a matching tag.
- `hn::LoadDup128(d, ptr)` — pointer must be aligned to 16 bytes.

### 8. Tag (`d`) declaration errors
